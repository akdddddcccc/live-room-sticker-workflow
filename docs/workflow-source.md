# 直播间贴片生图工作流（新对话迁移版）

## 1. 核心目标

用于在新对话中快速继续生成直播间贴片素材。

每一轮默认生成三张：

- 上贴：直播间顶部横向贴片
- 侧贴：直播间侧边竖向贴片
- 底贴：直播间底部横向贴片

当前流程只做背景、材质、留白、渐隐、装饰语言，不在生图阶段融合正式文字、logo、二维码、人物、产品或完整海报排版。

---

## 2. 每轮固定原则

### 2.1 一轮三图

用户给参考图后，不用反复询问，直接按同一风格生成：

1. 上贴
2. 侧贴
3. 底贴

除非用户明确说“只出一张”或“只重出某一张”。

### 2.2 严格跨轮隔离

每一轮只继承当前参考图，不继承上一轮任何风格残留。

必须隔离：

- 色彩
- 材质
- 光效
- 纹理
- 构图气质
- 装饰元素
- 提示词里的旧词汇

如果新参考图与上一轮相近，也要明确区分。例如：

- 红橙纸纹 ≠ 红金书法笔触
- 薄荷绿水彩雾景 ≠ 单色灰青水彩叠染
- 科技线稿 ≠ 参数化银灰波纹
- 地标建筑 ≠ 抽象空间流线

### 2.3 只提取视觉语言

参考图只用于提取：

- 构图节奏
- 色彩关系
- 材质质感
- 光效气质
- 装饰密度
- 留白方式
- 边缘元素分布

不要复制：

- 文字
- logo
- 品牌
- 标题
- 日期
- 页面按钮
- 二维码
- 完整海报版式
- 人物、手、产品、动物
- 明确主体，除非用户明确要求保留意向

---

## 3. 三类贴片结构规则

### 3.1 上贴

上贴用于直播间顶部。

固定规则：

- 16:9 横向
- 装饰主要在上沿、左右上角、左右边缘
- 中心不一定要浅色或纯白，只要视觉安静即可
- 中心应低纹理、低复杂度、低干扰，可叠加后期信息
- 底边必须保留明确、干净、柔和的中性纯白渐变
- 底部纯白渐变通常占 20%-28%
- 最底边不能有复杂线条、深色块、强纹理或主体元素

注意：上贴中心可以延续当前轮主色，例如深红、灰蓝、银灰、淡绿等；核心是“不复杂”，不是“一定白”。

### 3.2 侧贴

侧贴用于直播间侧边。

固定规则：

- 竖向窄幅
- 装饰默认只放左上角或上边沿
- 其余 75%-88% 区域保持素净、弱纹理、低对比
- 侧贴不强制向纯白过渡
- 侧贴可保留当前轮同色系安静底色
- 不要下角强装饰
- 不要贯穿整条边的复杂纹理
- 不要做满版插画或海报边框

侧贴判断标准：一眼看过去应该是“辅助边缘”，不是一张独立主视觉。

### 3.3 底贴

底贴用于直播间底部。

固定规则：

- 16:9 横向
- 视觉重心集中在下沿、左右下角、底部边缘
- 顶边必须保留明确、干净、柔和的中性纯白渐变
- 顶部纯白渐变通常占 25%-32%
- 最顶边不能有复杂线条、深色块、强纹理或主体元素
- 底部装饰应比上贴更稳、更低、更克制
- 不要硬底条、页脚栏、信息栏感

---

## 4. 焦点与直播间主体关系

直播间主体通常在画面中间，所以带空间纵深或聚焦感的图要注意方向：

- 上贴：纵深焦点向下，朝直播间中心汇聚
- 底贴：纵深焦点向上，朝直播间中心汇聚
- 侧贴：不做强纵深，只做边缘辅助锚点

示例：

```text
上贴的透视焦点位于画面下方中部，光线从上沿和左右上角向直播间中心收束。
底贴的透视焦点位于画面上方中部，光线从下沿和左右下角向直播间中心收束。
```

## 5. 常用提示词结构

每一轮按这个结构组织：

```text
轮次目标：参考当前唯一图片，生成直播间贴片三件套中的 <上贴/侧贴/底贴> 底图。只做背景视觉语言，不融合文字。

当前唯一参考图：<描述参考图视觉语言>。
只提取：<构图、色彩、材质、光效、装饰密度>。
不要复制：<文字、logo、主体、海报版式等>。

严格风格隔离：
- 不要上一轮的 <色彩/材质/光效/元素>
- 不要更早轮次的 <典型残留>
- 当前轮只使用 <本轮核心视觉语言>

贴片类型：<上贴/侧贴/底贴>
输出用途：<直播间顶部/侧边/底部贴片底图>

构图要求：
- <边缘装饰位置>
- <中心或内侧是否安静>
- <上贴底边/底贴顶边纯白渐变要求>
- <侧贴装饰收敛要求>

色彩要求：
- 主色：<当前轮主色>
- 辅助色：<当前轮辅助色>
- 必须色场：<上贴底边纯白 / 底贴顶边纯白 / 侧贴不强制纯白>
- 禁用色：<上一轮或不想要的颜色>

材质与质感：
- <当前轮材质>
- <光效/颗粒/线条/块面/水彩/玻璃等>

负向约束：
- 不要文字、字母、数字、logo、水印、二维码
- 不要人物、产品、动物、角色
- 不要信息图、UI、卡片、海报模板、促销元素
- 不要硬边框、硬底条
- 不要任何过往轮次风格残留
```

## 6. 风格转译经验

### 6.1 主体要转译，不要照搬

如果参考图有明显主体，先判断用户是否要保留意向。

示例：

- 马形金色笔触：不要马，转译为金色书法飞白
- 花影：不要花，转译为柔焦色雾
- 山林水彩：不要山林，转译为薄荷绿水彩雾层
- 科技线稿：不要紫色原图，换色后只保留工程弧线和网格
- 树：如果用户要树的意向，转译为抽象 low poly 生命树、分形枝干、分子节点
- 地标：可保留为抽象白色浮雕、几何剪影、结构线，不做旅游海报

### 6.2 红色高级感

红色要高级，避免鲜红、金红、橙红促销感。

推荐：

- 勃艮第红
- 黑樱桃红
- 烟灰玫瑰红
- 暗红玻璃
- 珍珠白高光
- 冷银反光

不要：

- 大面积正红
- 红金节庆组合
- 金币、红包、礼盒
- 高饱和橙红
- 密集舞台灯效

### 6.3 侧贴必须克制

侧贴常见问题是太花。

默认修正：

```text
侧贴装饰只集中在左上角和上边沿。
其余 80% 以上区域保持素净、弱纹理、低对比。
不需要强制向白色过渡。
不要下角装饰，不要整条边复杂纹理。
```

### 6.4 纯白渐变属于结构，不属于风格继承

上贴底边、底贴顶边的纯白渐变是贴片结构要求。

这不算继承上一轮风格。

但纯白渐变不能带入上一轮的材质，例如：

- 不要“纸纹白”污染科技轮
- 不要“淡杏白”污染红金轮
- 不要“水彩纸白”污染金属轮

正确表达：

```text
中性纯白渐变，干净、无线条、无纹理、无色块干扰。
```

## 7. Figma 素材库步骤

如果有 Figma 权限，需要把生成图沉淀到素材库。

当前指定文件：

```text
https://www.figma.com/design/2FpPZLxtKEpQg3pVSzF8rJ/Untitled?node-id=49-2&t=6FIVK3LQLiAD91rU-1
```

工作方式：

1. 生成每轮三件套
2. 上传到 Figma 文件
3. 按轮次分组摆放
4. 每组标注风格关键词，例如：
   - 银灰参数化波纹
   - 深绿园林矩形拼贴
   - low poly 科学生命树
   - 高级红色空间聚焦
5. 上贴、侧贴、底贴横向排列

注意：

- Figma 上传可能受 Starter plan 工具调用额度限制
- 如果额度不足，先在聊天中保留生成结果，额度恢复后再批量上传
- 单次上传最多 5 张

---

## 8. 质量检查清单

生成后检查：

- 是否每轮出了上贴、侧贴、底贴三张
- 是否只继承当前参考图
- 是否有上一轮颜色、材质或提示词残留
- 是否误复制了参考图主体
- 是否出现文字、logo、二维码、水印
- 是否变成海报、信息图、UI 或促销物料
- 上贴底边是否有纯白渐变
- 底贴顶边是否有纯白渐变
- 侧贴是否足够素净
- 中心/内侧是否可叠加
- 是否有硬边框、硬底条、页脚感
- 色彩是否高级、克制，不俗气
